What is DKIM?

DKIM adds a digital signature so receivers can verify the sender’s domain.

How DKIM Works

The sender signs headers; the receiver verifies using a public key in DNS.

Selectors & Keys

A selector (s=) tells which DNS key to use, enabling safe key rotation.

Why DKIM Matters

It detects tampering and helps protect brand trust and message integrity.

DKIM in Digital Forensics

Results like “dkim=pass/fail” can support spoofing/tampering analysis.

DKIM • Deep Dive

Understand integrity and authenticity

DKIM signs selected headers (and sometimes the body) with a private key. Receivers verify the signature using the public key published in DNS.

What you’ll see in headers
DKIM-Signature: v=1; a=rsa-sha256; d=example.com; s=selector1; ...

DKIM verification outcomes

Receivers validate the signature and report results such as pass or fail. DKIM alone doesn’t block mail — it provides cryptographic proof for trust decisions (often combined with DMARC).

✓ dkim=pass
✕ dkim=fail
! signature missing
⟳ key rotated

DKIM in practice

What students usually miss

These DKIM concepts show up often in investigations and quiz questions.

Signing vs Encryption

DKIM is a signature, not encryption. It proves integrity and domain-level authenticity, but it doesn’t hide the message content.

Quiz-aligned

Selectors (s=) & DNS keys

The selector points to a DNS record holding the public key. This lets senders rotate keys without breaking mail. Forensics often checks if the selector/key exists and matches.

Common troubleshooting

Forensics (auth results)

Investigators correlate DKIM results (e.g., Authentication-Results) with other signals like SPF/DMARC to determine spoofing vs legitimate sending infrastructure.
